Widget
	derives from Object

METHODS

*
xguimpl::Widget * getImpl()
	- returns a pointer to the widget implementation class

*
void setAsFloating()
	- unlink the widget from its parent

*
xgui::Container * getParent()
	- returns the widget parent container

*
xgui::Window * getContext()
	- returns the widget context (i.e.: the top-level window that contains it)

*
std::string getPath()
	- returns a string representing the widget and all its parents up to the containing top-level window;
	  the returned string has the following format:
	  	<top_level_window_id>/<parent_id>[/<parent_id> ...]/<widget_id>

*
bool linkEvent( std::string const &name, xgui::Callback * cb )
	- links a callback method to the specified event <name>; this method returns false if the
	  event could not be found

*
xgui::Callback * getEvent( std::string const &name )
	- returns a callback linked to the specified event <name>

*
bool hasEvent ( std::string const &name )
	- returns true if the specified event <name> is linked

*
void recalcLayout()
	- recalculates the dimensions and positions of the widget and all its children

*
std::pair<int, int> getMousePos()
	- returns a <x, y> pair that specifies the mouse position relative to the widget upper left corner

*
void redraw()
	- forces a redraw of the widget


PROPERTIES

+
enabled (RW)
	- enables or disables mouse and keyboard input to the widget; set the property value to "1" to enable, "0" to disable
	  default: "1"

+
tooltip-text (RW)
	- sets or retrieves the text of the widget tooltip; if set to empty string no tooltip will be shown
	  default: ""

+
visible (RW)
	- if set to "1" the widget is visible, if set to "0" the widget is hidden
	  default: "1"

+
focused (RW)
	- sets or removes the keyboard focus to the widget; if set to "1" the widget has focus, "0" otherwise


+
width (RO)
	- gets the widget width in pixels


+
height (RO)
	- gets the widget height in pixels


+
path (RO)
	- retrieves the path of the widget (the same returned by the method getPath())


EVENTS

@
ondestroy (linked to xgui::Callback)
	- fired when the widget is destroyed


@
onfocus (linked to xgui::Callback)
	- fired when the widget gets keyboard focus


@
onblur (linked to xgui::Callback)
	- fired when the widget looses keyboard focus


@
onmousedown (linked to xgui::TextStatusCallback)
	- fired when the user presses a mouse button while the cursor is in the client area of the widget
	  the string parameter specifies which button was clicked:
	  	it can be "left" for left-button, "right" for right-button or "middle" for middle-button
	  the status parameter can be one or more of the following values:
	  	xgui::mouseLeftMask		- the left mouse button is pressed
		xgui::mouseMiddleMask		- the middle mouse button is pressed
		xgui::mouseRightMask		- the right mouse button is pressed
		xgui::keyControlMask		- the control key is pressed
		xgui::keyShiftMask		- the shift key is pressed
		xgui::keyAltMask		- the alt key is pressed


@
onmouseup (linked to xgui::TextStatusCallback)
	- fired when the user releases a mouse button while the cursor is in the client area of the widget
	  the string parameter specifies which button was clicked:
	  	it can be "left" for left-button, "right" for right-button or "middle" for middle-button
	  the status parameter can be one or more of the following values:
	  	xgui::mouseLeftMask		- the left mouse button is pressed
		xgui::mouseMiddleMask		- the middle mouse button is pressed
		xgui::mouseRightMask		- the right mouse button is pressed
		xgui::keyControlMask		- the control key is pressed
		xgui::keyShiftMask		- the shift key is pressed
		xgui::keyAltMask		- the alt key is pressed


@
onmouseclick (linked to xgui::TextStatusCallback)
	- fired when the user clicks (i.e.: presses and releases) a mouse button while the cursor is in the client area of the widget
	  the string parameter specifies which button was clicked:
	  	it can be "left" for left-button, "right" for right-button or "middle" for middle-button
	  the status parameter can be one or more of the following values:
	  	xgui::mouseLeftMask		- the left mouse button is pressed
		xgui::mouseMiddleMask		- the middle mouse button is pressed
		xgui::mouseRightMask		- the right mouse button is pressed
		xgui::keyControlMask		- the control key is pressed
		xgui::keyShiftMask		- the shift key is pressed
		xgui::keyAltMask		- the alt key is pressed


@
onmousedblclick (linked to xgui::TextStatusCallback)
	- fired when the user double-clicks a mouse button while the cursor is in the client area of the widget
	  the string parameter specifies which button was clicked:
	  	it can be "left" for left-button, "right" for right-button or "middle" for middle-button
	  the status parameter can be one or more of the following values:
	  	xgui::mouseLeftMask		- the left mouse button is pressed
		xgui::mouseMiddleMask		- the middle mouse button is pressed
		xgui::mouseRightMask		- the right mouse button is pressed
		xgui::keyControlMask		- the control key is pressed
		xgui::keyShiftMask		- the shift key is pressed
		xgui::keyAltMask		- the alt key is pressed


@
onmousemove (linked to xgui::TextCallback)
	- fired when the user moves the cursor over the client area of the widget
	  the string parameter specifies the cursor position relative to the widget upper left corner
	  in the format "x y"


@
onmouseout (linked to xgui::Callback)
	- fired when the cursor leaves the client area of the widget

	
@
onmouseover (linked to xgui::Callback)
	- fired when the cursor enters the client area of the widget


@
onhide (linked to xgui::Callback)
	- fired when the widget becomes hidden


@
onshow (linked to xgui::Callback)
	- fired when the widget becomes visible

@
onenable (linked to xgui::Callback)
	- fired when the widget becomes enabled (i.e: it accepts mouse and keyboard input)


@
ondisable (linked to xgui::Callback)
	- fired when the widget becomes disabled (i.e: it ignores mouse and keyboard input)


@
onresize (linked to xgui::TextCallback)
	- fired when the widget size has changed
	  the string parameter specifies the new size of the widget in the format "width height"

@
onmove (linked to xgui::TextCallback)
	- fired when the widget position has changed
	  the string parameter specifies the new position of the upper-left corner of the widget
	  in the format "x y"; the position is relative to the widget parent

@
onkeydown (linked to xgui::TextCallback)
	- fired when the user presses a key while the widget has keyboard focus
	  the string parameter specifies the pressed key


@
onkeyup (linked to xgui::TextCallback)
	- fired when the user releases a key while the widget has keyboard focus
	  the string parameter specifies the released key

@
onkeypress (linked to xgui::TextCallback)
	- fired when the user presses and releases a key while the widget has keyboard focus
	  the string parameter specifies the released key
